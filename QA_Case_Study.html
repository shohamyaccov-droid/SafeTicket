The Bug

Problem: Race Condition in Group Purchases

When buyers attempted to purchase multiple tickets from a listing group, the backend checked the specific displayed ticket's status first. If that ticket was already sold, the system immediately returned "Ticket is no longer available" - even though other tickets in the same group were still available.

Root Cause

# PROBLEMATIC CODE
def create_order(request):
    ticket_id = request.data.get('ticket_id')
    listing_group_id = request.data.get('listing_group_id')
    
    # ❌ Check ticket_id status FIRST
    ticket = Ticket.objects.get(id=ticket_id)
    if ticket.status != 'active':
        return error("Ticket is no longer available")  # Fails here!
    
    # This code never reached if ticket_id is sold
    if listing_group_id:
        # Find other tickets in group...

Impact: 15-20% transaction failure rate, customer complaints, lost revenue.

AI-Assisted Rapid Solution

The Fix

When listing_group_id is provided, completely ignore the specific ticket_id status and search for ANY available tickets within that group.

Before

# Check ticket_id first
ticket = Ticket.objects.get(id=ticket_id)
if ticket.status != 'active':
    return error("Not available")
# Group check never reached

After

# Check group FIRST
if listing_group_id:
    available_tickets = Ticket.objects.filter(
        listing_group_id=listing_group_id,
        status__in=['active', 'reserved']
    ).order_by('id')[:quantity]
    # Use any available tickets
